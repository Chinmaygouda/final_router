STEP 3: Setup PostgreSQL
$ createdb semantic_vault

STEP 4: Initialize database
$ python -c "from app.database_init import initialize_v2_db; initialize_v2_db()"

$ docker run -d -p 6379:6379 redis:latest
OR
$ redis-server

STEP 6: Start FastAPI
$ uvicorn app.main:app --reload


# ==================== 2. BASIC USAGE ====================

PYTHON EXAMPLE:
```python
import requests

url = "http://localhost:8000/ask"
payload = {
    "user_id": "user_123",
    "prompt": "What is machine learning?",
    "user_tier": 1
}

response = requests.post(url, json=payload)
print(response.json())
```

CURL EXAMPLE:
```bash
curl -X POST http://localhost:8000/ask \
  -H "Content-Type: application/json" \
  -d '{
    "user_id": "user_123",
    "prompt": "What is machine learning?",
    "user_tier": 1
  }'
```

JAVASCRIPT EXAMPLE:
```javascript
const payload = {
    user_id: "user_123",
    prompt: "What is machine learning?",
    user_tier: 1
};

fetch('http://localhost:8000/ask', {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify(payload)
})
.then(r => r.json())
.then(data => console.log(data));
```

# ==================== 6. VAULT SEMANTICS ====================

HOW SEMANTIC CACHING WORKS:

Request 1: "What is machine learning?"
└─ Generates embedding → Stores in vault
└─ Cost: $0.001 (API call)
└─ Source: AI_GENERATION

Request 2: "Tell me about machine learning"
└─ Similar embedding (L2 distance < 0.5)
└─ Vault HIT! Returns cached response
└─ Cost: $0.00 (no API call)
└─ Source: VAULT_CACHE

THRESHOLD: 0.5 (can be adjusted)
└─ 0.1 = almost identical
└─ 0.5 = similar (default)
└─ 0.8 = loosely related

Your vault persists across sessions:
└─ More questions → More cache hits
└─ More cache hits → Lower costs

# ==================== 10. TROUBLESHOOTING ====================

ERROR: "Could not connect to Docker container"
FIX: Redis is optional. System works without it (just slower)
$ docker run -d -p 6379:6379 redis:latest

ERROR: "DATABASE_URL not set"
FIX: Add DATABASE_URL to .env file
$ echo "DATABASE_URL=postgresql://localhost/semantic_vault" >> .env

ERROR: "401 Unauthorized" (API key error)
FIX: Check your API keys are correct and have access
$ grep OPENAI_API_KEY .env

ERROR: "No module named 'app'"
FIX: Make sure you're running from project root
$ cd "d:\final router"
$ python -c "from app.main import app"

ERROR: "pgvector: No such extension"
FIX: Enable pgvector extension
$ psql semantic_vault -c "CREATE EXTENSION IF NOT EXISTS vector;"

ERROR: Connection timeout
FIX: Verify PostgreSQL is running
$ psql -l

# ==================== 12. PRODUCTION DEPLOYMENT ====================

DOCKER COMPOSE (recommended):
```yaml
version: '3'
services:
  postgres:
    image: postgres:15
    environment:
      POSTGRES_DB: semantic_vault
      POSTGRES_PASSWORD: password
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:latest
    ports:
      - "6379:6379"

  app:
    build: .
    command: uvicorn app.main:app --host 0.0.0.0 --port 8000
    ports:
      - "8000:8000"
    environment:
      DATABASE_URL: postgresql://postgres:password@postgres/semantic_vault
    depends_on:
      - postgres
      - redis

volumes:
  postgres_data:
```

GUNICORN (production ASGI):
```bash
gunicorn app.main:app --workers 4 --worker-class uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
```

NGINX (reverse proxy):
```nginx
upstream app {
    server app:8000;
}

server {
    listen 80;
    location / {
        proxy_pass http://app;
    }
}
```
